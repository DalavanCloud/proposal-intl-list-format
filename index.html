Intl.ListFormat Specification Proposal

Proposal Details

Current Stage: 0 / Oct 2015

1ListFormat Objects#

1.1The Intl.ListFormat Constructor#

The ListFormat constructor is a standard built-in property of the Intl object. Behaviour common to all service constructor properties of the Intl object is specified in .

1.1.1InitializeListFormat (listFormat, locales, options)#

The abstract operation InitializeListFormat accepts the arguments listFormat (which must be an object), locales, and options. It initializes listFormat as a ListFormat object.

The following steps are taken:

  1. If listFormat has an [[initializedIntlObject]] internal slot with value true, throw a TypeError exception.
  2. Set listFormat.[[initializedIntlObject]] to true.
  3. Let requestedLocales be CanonicalizeLocaleList(locales).
  4. ReturnIfAbrupt(requestedLocales).
  5. If options is undefined, then
    1. Let options be ObjectCreate(%ObjectPrototype%).
  6. Else
    1. Let options be ToObject(options).
    2. ReturnIfAbrupt(options).
  7. Let opt be a new Record.
  8. Let matcher be GetOption(options, "localeMatcher", "string", «"lookup", "best fit"», "best fit").
  9. ReturnIfAbrupt(matcher).
  10. Set opt.[[localeMatcher]] to matcher.
  11. Let localeData be %ListFormat%.[[localeData]].
  12. Let r be ResolveLocale(%ListFormat%.[[availableLocales]], requestedLocales, opt, %ListFormat%.[[relevantExtensionKeys]], localeData).
  13. Set listFormat.[[locale]] to the value of r.[[locale]].
  14. Let dataLocale be r.[[dataLocale]].
  15. Let s be GetOption(options, "style", "string", «"regular", "duration", "duration-short", "duration-narrow", "number"», "regular").
  16. ReturnIfAbrupt(s).
  17. Set listFormat.[[style]] to s.
  18. Let dataLocaleData be Get(localeData, dataLocale).
  19. Let listFormat be Get(dataLocaleData, "listFormat").
  20. Assert: listFormat is an object (see 1.2.3).
  21. Set listFormat.[[listFormat]] to Get(listFormat, s).
  22. Set listFormat.[[boundResolve]] to undefined.
  23. Set listFormat.[[initializedListFormat]] to true.
  24. Return listFormat.

1.1.2Intl.ListFormat([ locales [, options]])#

When the Intl.ListFormat function is called with optional arguments the following steps are taken:

  1. If NewTarget is undefined, let newTarget be the active function object, else let newTarget be NewTarget.
  2. Let listFormat be OrdinaryCreateFromConstructor(newTarget, %ListFormatPrototype%).
  3. ReturnIfAbrupt(listFormat).
  4. Return InitializeListFormat(listFormat, locales, options).

1.2Properties of the Intl.ListFormat Constructor#

Besides the internal slots and the length property (whose value is 0), the Intl.ListFormat constructor has the following properties:

1.2.1Intl.ListFormat.prototype#

The value of Intl.ListFormat.prototype is %ListFormatPrototype%.

This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }.

1.2.2Intl.ListFormat.supportedLocalesOf (locales [, options ])#

When the supportedLocalesOf method of %ListFormat% is called, the following steps are taken:

  1. Let availableLocales be %ListFormat%.[[availableLocales]].
  2. Let requestedLocales be CanonicalizeLocaleList(locales).
  3. Return SupportedLocales(availableLocales, requestedLocales, options).

The value of the length property of the supportedLocalesOf method is 1.

1.2.3Internal slots#

The value of the [[availableLocales]] internal slot is implementation defined within the constraints described in .

The value of the [[relevantExtensionKeys]] internal slot is ["pr"].

The value of the [[localeData]] internal slot is implementation defined within the constraints described in and the following additional constraints:

  • [[localeData]][locale] must have a listFormat property for all locale values. The value of this property must be an object, which must have properties with the names of the five list formatting styles: "regular", "duration", "duration-short", "duration-narrow" and "number". Properties "regular", "duration", "duration-short" and "duration-narrow" must be objects themselves, and each must have properties with names of the templates: "2", "start", "middle" and "end. Each of those properties must be a template string as specified in LDML List Format Rules. Property "number" must have a string value with a corresponding template string as specified in LDML List Format Rules.
Note It is recommended that implementations use the locale data provided by the Common Locale Data Repository (available at http://cldr.unicode.org/).

1.3Properties of the Intl.ListFormat Prototype Object#

The Intl.ListFormat prototype object is itself an Intl.ListFormat instance as specified in 1.4, whose internal slots are set as if it had been constructed by the expression Construct(%ListFormat%).

In the following descriptions of functions that are properties or [[Get]] attributes of properties of %ListFormatPrototype%, the phrase "this ListFormat object" refers to the object that is the this value for the invocation of the function; a TypeError exception is thrown if the this value is not an object or an object that does not have an [[initializedListFormat]] internal slot with value true.

1.3.1Intl.ListFormat.prototype.constructor#

The initial value of Intl.ListFormat.prototype.constructor is %ListFormat%.

1.3.2Intl.ListFormat.prototype[@@toStringTag]#

The initial value of the @@toStringTag property is the string value "Object".

This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: true }.

1.3.3Intl.ListFormat.prototype.format#

This named accessor property returns a function that formats a list according to the effective locale and the formatting options of this ListFormat object.

The value of the [[Get]] attribute is a function that takes the following steps:

  1. Let pf be this ListFormat object.
  2. If pf.[[boundResolve]] is undefined, then
    1. Let F be a new built-in function object as defined in 1.3.4.
    2. The value of F’s length property is 1.
    3. Let bf be BoundFunctionCreate(F, «this value»).
    4. Set pf.[[boundResolve]] to bf.
  3. Return pf.[[boundResolve]].
Note The function returned by [[Get]] is bound to this ListFormat object so that it can be passed directly to Array.prototype.map or other functions.

The value of the [[Set]] attribute is undefined.

1.3.4List Format Functions#

A ListFormat format function is an anonymous function that takes one argument value, and performs the following steps:

  1. Let listFormat be the this value.
  2. Assert: Type(listFormat) is Object and listFormat has an [[initializedListFormat]] internal slot whose value is true.
  3. If value is not provided or is undefined, then
    1. Let value be a new empty List.
  4. Else
    1. Let value be CreateArrayFromList(value)
  5. Return FormatList(listFormat, value).

When the ListFormat abstract operation is called with arguments listFormat (which must be an object initialized as a ListFormat) and value (which must be a List value), it returns a String value representing the formatted list of elements in value according to the effective locale and the options of listFormat.

The computations rely on String values and locations within numeric strings that are dependent upon the implementation and the effective locale of listFormat (“ILD") or upon the implementation. The ILD String mentioned, must not contain any characters in the General Category “Number, decimal digit" as specified by the Unicode Standard.

Note It is recommended that implementations use the locale data provided by the Common Locale Data Repository (available at http://cldr.unicode.org/).

The following steps are taken:

  1. Let formatListFunction be the value of listFormat.[[listFormat]].
  2. Let result be formatListFunction(n).
  3. Return result.

1.3.5Intl.ListFormat.prototype.resolvedOptions ()#

This function provides access to the locale and options computed during initialization of the object.

The function returns a new object whose properties and attributes are set as if constructed by an object literal assigning to each of the following properties the value of the corresponding internal slot of this ListFormat object (see 1.4): locale and style. Properties whose corresponding internal slots are not present are not assigned.

1.4Properties of Intl.ListFormat Instances#

Intl.ListFormat instances inherit properties from %ListFormatPrototype%.

Intl.ListFormat instances and other objects that have been successfully initialized as a ListFormat have [[initializedIntlObject]] and [[initializedListFormat]] internal slots whose values are true.

Objects that have been successfully initialized as a ListFormat object also have several internal slots that are computed by the constructor:

  • [[locale]] is a String value with the language tag of the locale whose localization is used by the list format styles.
  • [[style]] is one of the String values "regular", "duration", "duration-short", "duration-narrow" or "number", identifying the list formatting style used.

Finally, objects that have been successfully initialized as a ListFormat have a [[boundResolve]] internal slot that caches the function returned by the format accessor (1.3.3).
